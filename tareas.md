# 📝 Tareas del Día — Tu Portafolio Personal

> **GitHub Copilot Dev Days | Ciudad del Este** · 25 de Abril 2026
> Marcá cada tarea a medida que la completás. El foco principal está en la mañana.

---

## 🎯 Objetivo del Día

> Al terminar el evento, **tendrás un portafolio personal desplegado en internet**, con tu código en GitHub y tu URL pública viva en Vercel.

---

## 🌅 MAÑANA (09:00 – 12:00)

### 🧭 Objetivo macro
Tener una **base funcional del portafolio corriendo en `localhost`**, con el código ya versionado en GitHub y el primer push realizado.

---

### 🔐 Bloque 1 — Cuenta y repositorio GitHub

- [ ] **1.** Crear cuenta en **GitHub** (si todavía no tenés) → [github.com](https://github.com)
- [ ] **2.** Crear un repositorio **público** llamado `mi-portafolio`
- [ ] **3.** Marcar la opción *"Add a README file"* al crearlo
- [ ] **4.** Copiar la URL del repo (la vas a usar ahora)

---

### 💻 Bloque 2 — Entorno local

- [ ] **5.** Abrir **VS Code**
- [ ] **6.** Clonar el repo: `git clone <url-del-repo>`
- [ ] **7.** Abrir la carpeta clonada en VS Code
- [ ] **8.** Instalar la extensión **Live Server** (para el preview local)

---

### 📁 Bloque 3 — Estructura del proyecto

Crear los 4 archivos base (vacíos por ahora):

- [ ] **9.** `index.html`
- [ ] **10.** `style.css`
- [ ] **11.** `script.js`
- [ ] **12.** Actualizar el `README.md` con una descripción breve

---

### 🤖 Bloque 4 — Copilot en acción

- [ ] **13.** Abrir **Copilot Chat** (ícono lateral o `Ctrl + Alt + I`)
- [ ] **14.** Cargar los archivos con `#` como contexto (ej: `#index.html`)
- [ ] **15.** Explorar el **selector de modelos LLM** (GPT / Claude / Gemini)
- [ ] **16.** Pedir a Copilot la **estructura base del HTML** con 5 secciones: *Hero, Sobre mí, Proyectos, Skills, Contacto*
- [ ] **17.** Pedir el **CSS inicial** (variables, reset, tipografía, colores base)
- [ ] **18.** **Personalizar con tus datos reales** (tu nombre, tu rol, tu bio, tus proyectos)

---

### ✅ Bloque 5 — Preview y primer push

- [ ] **19.** Click derecho en `index.html` → **"Open with Live Server"**
- [ ] **20.** Verificar que se ve el portafolio en tu navegador
- [ ] **21.** En la terminal de VS Code: `git add .`
- [ ] **22.** `git commit -m "primer portafolio base"`
- [ ] **23.** `git push origin main`
- [ ] **24.** Verificar en GitHub.com que tu código esté arriba

---

### 🏁 Criterios de éxito (antes del almuerzo)

Al cerrar la mañana deberías tener marcados:

- ✅ Repo público en GitHub con el código pusheado
- ✅ Portafolio renderizando en el navegador (`localhost`)
- ✅ Mínimo **3 secciones visibles con tus datos reales**
- ✅ Copilot Chat activo y funcionando con `#`

> Si llegaste acá antes de las 12:00 → **seguí iterando** con Copilot: agregá más proyectos, mejorá los colores, sumá animaciones.

---

## 🍽️ ALMUERZO (12:00 – 13:30)

Libre. **Plaza City** tiene varios locales gastronómicos dentro y en los alrededores. Ideal para seguir haciendo networking.

---

## 🌆 TARDE (13:30 – 17:00)

### 🧭 Objetivo macro
**Documentar el proyecto con un PRD y desplegarlo público en Vercel** con URL real que puedas compartir.

### 📋 Checklist resumido

- [ ] Asistir a la **presentación del PRD** (13:30 – 14:15)
- [ ] Escribir tu propio **`PRD.md`** usando la plantilla + Copilot
- [ ] Commit del `PRD.md` al repo de GitHub
- [ ] Iterar features del portafolio con Copilot (más proyectos, estilos, micro-animaciones)
- [ ] Crear cuenta en **Vercel** (con tu cuenta de GitHub → login rápido)
- [ ] **Conectar tu repo** `mi-portafolio` a Vercel
- [ ] Deploy automático → obtener tu URL `tu-nombre.vercel.app`
- [ ] Hacer un nuevo push y **verificar que Vercel redeploya solo**
- [ ] **Compartir tu URL** con el grupo y con la comunidad

### 🏁 Criterios de éxito (al final del evento)

- ✅ Portafolio **en vivo con URL pública** de Vercel
- ✅ `PRD.md` commiteado al repo
- ✅ Mínimo **5 secciones completas y personalizadas**
- ✅ **Cada push a GitHub redeploya automáticamente** en Vercel
